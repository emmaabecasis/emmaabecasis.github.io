Random Forests

Random forests can be used for either classification or regression problems. Random forests use many decision trees to make an overall decision about an input value.

Decision Trees

1

Decision trees break down the input features into separate regions to make predictions. It maps an input vector x to a predicted output or y value. The prediction function for a decision tree is:

  • T: number of terminal nodes/leaves in the tree (these represent the end of a decision path)
  • Cm: the prediction value assigned to Rm (the most common class in classification problems or the mean of the target variable in regression problems)
  • Rm: the m-th region (leaf) of the feature space (a specific set of features)
  • :indicator function that equals 1 if x belongs to region Rm and 0 if not

Essentially, for a given input vector x, the tree will go down its structure until reaching a terminal node where the prediction cm is made based on the values in that region of features.

Regression Trees

Decision trees output a continuous value by averaging the values of the target variable for all training samples in a given region.

Tree Construction

Decision trees use a specific splitting criterion to determine how to split data at each node.

  • Gini index (classification): measures impurity at a given node with a lower value indicating a purer node. The index represents the probability that a randomly selected sample would be misclassified if it were labeled according to the distribution at a node. In the following equation for the Gini index, pk is the proportion of samples in class k at node t.
  • Entropy (classification): measures uncertainty in class distribution at a given node. A lower value means that a node contains mostly one class. This value quantifies the amount of information needed to classify a new sample. In the equation for entropy, pk is the proportion of samples in class k.
  • Variance reduction (regression): splits in regression trees are chosen to minimize the variance of the target variable at each node. This minimizes the prediction error. Implementation

Regression:

We will start with a random forest regression example using Scikit-learn. You can follow along on Colab or another platform.

  1. First, we import the necessary packages from Scikit-learn. The linnerud dataset contains 3 physiological variables (the targets) and 3 exercise variables (the explanatory variables) collected from 20 middle-aged men in a fitness club.4
  2. Next, we will load the dataset into two arrays. The X array contains the 3 exercise variables and the y array contains the 3 physiological variables. We only want to predict one of the physiological variables, so we will select only one from the y array.
  3. Now we can use the train_test_split function to split the dataset into the training and testing sets. The test set contains 20% of the total data (set by the test_size parameter). The random_state parameter ensures the same data will be put into the test set each time the line of code is run.
  4. We will now initialize the random forest regressor and fit it to the training data. We are trying to predict a physiological variable from exercise variables for 20 middle-aged men. Using the model, we will make predictions on the test set and store these in y_pred.

    When initializing the model, you can set many different hyperparameters to fine tune the model, such as the number of trees, the criterion for evaluating the quality of splits, the maximum depth of the tree, and more that can be found in the Scikit-learn documentation for a Random Forest Regressor.
  5. Finally, we will evaluate the model using the mean squared error and r-squared. The r-squared value for this dataset is negative, indicating that the regression model is a bad fit for this data. This could be because the exercise data is not a good predictor for the physiological variable or there is too little training data for the model to perform well.

Useful Applications

Random forests are robust models commonly used for both regression and classification problems. Some specific applications are:

  • Healthcare: random forests are helpful for making early diagnoses and can identify more complex patterns than linear models. They are also cheaper and easier to interpret than neural networks, which is a complex form of machine learning.
  • Stock market: recursive partitioning can divide stocks into groups based on their market value and compare returns on investment.
  • E-commerce: predict whether user will buy a product after viewing an ad.7
